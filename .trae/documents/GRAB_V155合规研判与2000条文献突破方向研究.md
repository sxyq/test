## 三、V155 DUAL-PRUNE 合规研判

### 3.1 V155 实现分析（来自 [V155 infer.py](file:///Users/sunyiyang/Desktop/Project/Baidu%20%20GRAB/versions/55_v155/V155-DUAL-PRUNE-sxyq/infer.py)）

V155 包含**两套剪枝**：

#### 剪枝 A：V153 PRUNE_FFN（SMoE 专家隐层维度剪枝）
- **位置**：infer.py 第 682-713 行
- **机制**：计算每个隐层神经元的重要性 `||fc1[row]|| * ||fc2[:, col]||`，保留 top-k 神经元
- **效果**：dim_ff 从 1024→768（25% 剪枝）
- **结构影响**：8 个 Expert 结构完整保留（输入→隐层→输出），仅隐层变窄

#### 剪枝 B：V155 PRUNE_HEAD_DIM（注意力头维度剪枝）
- **位置**：infer.py 第 1055-1089 行
- **机制**：计算每个 head_dim 索引的重要性 `||qkv_w|| * ||out_w||`，保留 top-k 维度
- **效果**：head_dim 从 64→48（25% 剪枝）
- **结构影响**：**改变了注意力的核心架构参数**

### 3.2 合规判定

| 剪枝项 | 合规判定 | 理由 |
|--------|----------|------|
| **V153 FFN 神经元剪枝** | ✅ **合规** | 标准结构化剪枝，移除冗余神经元，Expert 结构（8专家/Top-2门控）完整保留，符合"删掉冗余度较高、结构上可安全移除的部分" |
| **V155 head_dim 剪枝** | ⚠️ **高风险/潜在违规** | head_dim 是注意力机制的核心架构参数（d_model/n_heads=64），改变它意味着：①注意力缩放因子改变（√64→√48）；②QK^T 矩阵维度改变；③每个头的表征能力改变。这超出"安全移除冗余部分"范畴，属于**改变模型组网参数** |

### 3.3 判定依据

1. **规则字面**："不可修改模型组网和相关参数"——head_dim=64 是模型组网参数（d_model=512, n_heads=8 → head_dim=64），V155 将其改为 48，属于修改组网参数。

2. **学术共识**（来自在线研究）：
   - LASP 论文（ICLR 2026 投稿）明确指出："we do not recommend pruning sub-dimensions of Q/K/V"（不建议剪枝 Q/K/V 的子维度），因为这会导致与位置编码机制不一致
   - 标准的"注意力头剪枝"指移除整个 head（8→6 个头），而非缩小每个 head 的维度
   - CORP 论文（ICLR 2026）虽然剪枝 attention 维度，但需要**闭式补偿**（ridge regression 恢复表征），V155 无此补偿

3. **历史判例**：V116-V131 跳层被判违规，因为"破坏结构"。head_dim 剪枝同样改变了注意力结构，风险类似。

4. **PCOC 风险**：head_dim 改变会直接影响注意力分布，进而影响 logit 校准，PCOC 可能偏移出 [0.85,1.15] 区间。

### 3.4 建议

- **V153 FFN 剪枝**：可继续使用，配合 bias 烘焙修正 PCOC，属于合规优化路径
- **V155 head_dim 剪枝**：**不建议提交**，高风险被判违规导致 0 分。若要剪枝注意力，应改为**整头剪枝**（移除冗余的 attention head，如 8→6 个头），这在学术界是公认的结构化剪枝
- **更安全的替代**：用 SVD 低秩分解替代 head_dim 剪枝，数学上更严谨

### 4.2 关键发现：最具突破潜力的方向

基于 2000+ 条资源研究，按"合规性 × 突破概率 × V100 适用性"排序：

#### 🥇 突破方向 1：MegaBlocks/ScatterMoE 块稀疏 MoE 内核

| 维度 | 评估 |
|------|------|
| **来源** | MegaBlocks (MLSys 2023)、ScatterMoE (arXiv 2403.08245)、SonicMoE (ICLR 2026)、PyTorch Triton grouped GEMM (2025/08) |
| **原理** | 将 MoE 的动态路由计算表达为块稀疏操作，用 grouped GEMM 内核替代 padding 浪费 |
| **当前痛点** | V109 Dense SMoE 存在 **4x 过计算**（8 个 Expert 全算但只用 2 个），这是合规优化最大空间 |
| **预期加速** | 1.3-2x（消除过计算） |
| **V100 兼容** | ✅ 块稀疏内核支持 SM70；Triton 支持 V100 |
| **合规性** | ✅ 数学等价，不改变 Top-2 路由语义 |
| **风险** | 低——实现复杂度中等，需验证内核可用性 |
| **关键证据** | PyTorch 官方 Triton grouped GEMM 在 H100 上 2.62x 加速；SonicMoE 在 Hopper 上 1.86x |

#### 🥈 突破方向 2：INT8 W8A8 量化（SmoothQuant 思路）

| 维度 | 评估 |
|------|------|
| **来源** | SmoothQuant (ICML 2023)、NVIDIA PTQ 博客 (2025/08)、vLLM INT8 文档 |
| **原理** | 离线将量化难度从激活迁移到权重（channel-wise smoothing），实现 W8A8 INT8 GEMM |
| **当前痛点** | V109/V139 已用 FP16，但 V100 的 **INT8 Tensor Core (DP4A)** 算力是 FP16 的 2-4x，未被利用 |
| **预期加速** | 1.5-2x（INT8 GEMM vs FP16 GEMM） |
| **V100 兼容** | ✅ V100 原生支持 INT8 DP4A（compute capability 7.0） |
| **合规性** | ✅ 量化是明确允许的优化 |
| **风险** | 中——PCOC 可能偏移 ±0.02，需校准；激活异常值需处理 |
| **关键证据** | SmoothQuant 在 OPT-175B 上 1.56x 加速；vLLM 支持 compute capability >7.5，V100 (7.0) 需验证 |

#### 🥉 突破方向 3：xformers memory_efficient_attention

| 维度 | 评估 |
|------|------|
| **来源** | Meta xformers、theneuralbase 验证报告 (2026/04) |
| **原理** | 分块计算注意力，避免物化完整 attention matrix，IO 感知 |
| **当前痛点** | V139 的 flash-attn 在 V100 (SM70) 上可能静默回退到 SDPA，未真正生效 |
| **预期加速** | 1.1-1.5x（相比当前 SDPA 路径） |
| **V100 兼容** | ✅ **xformers 明确要求 sm_70+**，V100 是最低支持门槛，最可靠 |
| **合规性** | ✅ 数学等价 |
| **风险** | 零——数学等价，最安全的注意力加速 |
| **关键证据** | theneuralbase 验证："xformers requires sm_70+ (V100/A100/RTX series)"；部分模型 40% 延迟改善 |

### 4.3 V100 (SM70) 兼容性关键发现

这是本次研究的**最重要硬件发现**，直接影响所有优化方向可行性：

| 技术 | V100 支持 | 说明 |
|------|----------|------|
| FlashAttention-2 官方内核 | ❌ 需 SM80+ | V139 安装的 flash-attn **可能静默回退到 SDPA**，需验证 |
| **xformers memory_efficient** | ✅ SM70 | **V100 上最可靠的 IO 感知注意力** |
| FlexAttention (Triton) | ✅ SM70 | torch.compile 生成 Triton 内核 |
| INT8 Tensor Core (DP4A) | ✅ SM70 | V100 原生支持，算力约 FP16 的 2-4x |
| FP8 | ❌ 需 SM90+ | V100 不支持，FP8 代码静默回退 FP32 |
| BF16 | ❌ 需 SM80+ | V100 不支持，只能用 FP16 |
| CUDA Graph | ✅ | 形状固定时可 capture |
| Triton 内核 | ✅ SM70 | 支持 V100 |
| MegaBlocks 块稀疏 | ✅ SM70 | 块稀疏 GEMM 支持 V100 |

**核心结论**：V139 的 flash-attn 可能未真正生效，**xformers 是 V100 上最可靠的注意力加速方案**，应优先验证。

## 六、假设与决策

### 6.1 关键假设
1. V100 (SM70) 是评测环境，所有方案必须验证 SM70 兼容性
2. flash-attn 在 V100 上可能未生效（需线上验证）

4. PCOC 校准可通过 bias 烘焙零开销修正（V120 已验证）

### 6.2 关键决策
1. **放弃 V155 head_dim 剪枝**——高风险违规，不值得赌
2. **保留 V153 FFN 剪枝**——合规且有效
3. **优先 xformers 而非 flash-attn**——V100 原生支持更可靠
4. **MoE 内核是最大突破点**——4x 过计算是合规优化最大空间
5. **INT8 量化是第二大突破点**——V100 DP4A 被忽视
