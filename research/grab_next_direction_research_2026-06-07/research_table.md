## 2026-06-08 V112 Batch Scheduling Update

V112 explores a different system layer: whole-batch execution order.  Since
V42/V99 render `predict.txt` from a `logid -> prob` mapping in test-logid order,
shape-sorted batch execution can be output-equivalent when predicted logids are
unique.  The gate explicitly stops duplicate predicted logids because dict
last-wins semantics would become execution-order dependent.

Local synthetic evidence:

- safe cases: `3/4`;
- duplicate stop cases: `1`;
- median safe adjacent-shape switch reduction: `93.67%`.

This does not prove latency improvement.  It only justifies a future A800
CUDA Graph / `torch.compile` / shape-cache scheduling gate after V110 real
shape profiling.
